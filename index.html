Шифрование
Веб-приложение, использующее несколько способов шифрования и дешифрования текстовых сообщений, в основе которых лежит использование метода перестановки.
Продолжить
Введение
Прежде чем начать работать с приложением, нужно разобраться с такими понятиями, как шифрование и дешифрование, а также что такое метод перестановки.
Шифрование
Процесс преобразования исходного текста сообщения в такой вид, который будет не понятен случайному наблюдателю.
Дешифрование
Обратный шифрованию процесс преобразования шифрованного текста в его исходный вид, который будет понятен получателю.
Метод перестановки
Строго заданный алгоритм, согласно которому символы исходного сообщения переставляются между собой.
Документация
Для корректной работы приложения необходимо ознакомиться с небольшим справочным материалом, предоставленным ниже.
  • Имеются две секции - одна для шифрования текстовых сообщений, другая для дешифрования. Каждая из них содержит по два текстовых поля. Поле слева отвечает за ввод исходных данных, поле справа - за вывод обработанного результата.
  • В приложении задействованы три способа шифрования текстовых сообщений, каждый из которых имеет свой собственный уровень сложности. Первые два уровня сложности - способы, которые используют реверсивный подход. Они являются одновременно шифраторами и дешифраторами. Третий по сложности уровень способ - шифрования и дешифрование с использованием матриц (двумерных массивов). С некоторыми небольшими, отличиями он тоже используется одновременно как шифратор и дешифратор.
  • Приложение не имеет строгих правил при вводе исходных данных. После попытки отправить введённые данные на обработку, произойдет проверка только на наличие этих самых данных. Если поле ввода оказалось пустым - оно подсветится красным цветом, иначе - зеленым цветом.
  • Важное уточнение, что при дешифровании текстового сообщения, необходимо указать тот способ, который использовался при шифрованнии данного текста. Если этого не сделать - приложение может работать некорректно, а результаты дешифрования - быть неверными.
Шифрование
Для получения результата необходимо ввести исходный текст, а затем начать обработку. Полученный результат в выводе позже можно будет вставить в дешифратор, который расположен ниже.
Дешифрование
Для получения результата необходимо ввести зашифрованный текст, а затем начать обработку. Зашифрованный текст для ввода можно получить, использовав шифратор, который находится выше.
Описание
Приложение использует 3 способа шифрования и дешифрования методом перестановки. Ниже рассмотрены все способы с использованием языка программирования JavaScript.
Простой способ
Принимая текстовое сообщение, первым делом разбиваем его на массив, затем проходимся через цикл по каждому отдельному слову. Каждое слово также разбивается на массив, переворачивается и обратно переводится в строку. После преобразования каждого слова, текущий массив переворачивается и приводится к строке.
getMinCrypt(message) {
  return message
    .split(' ')
    .map( (word) => {
      return word
        .split('')
        .reverse()
        .join('');
    })
    .reverse()
    .join(' ');
}
Средний способ
Практически идентичен предыдущему способу, но отличается тем, что слова внутри текстового сообщения не переворачиваются, а последовательно попарно перебираются. В каждой паре происходит простая перестановка - символы меняются местами друг с другом.
getMidCrypt(message) {
  return message
    .split(' ')
    .map( (word) => {
      const list = word.split('');

      for (let curr >= 0, next = 1; next < list.length; curr++, next++) {
        if (next % 2 === 1) {
          const currEl = list[curr];
          const nextEl = list[next];

          list[curr] = nextEl;
          list[next] = currEl;
        }
      }

      return list.join('');
    })
    .reverse()
    .join(' ');
}
Сложный способ
Данный способ значительно отличается от предыдущих двух, потому что в основе шифрования и дешифрования лежит использование матриц. Сообщение записывается в матрицу с фиксированной шириной, а после переворачивается и считывается посторочно. Данный способ также реализован с помощью одной функции, которая выполняет роль как шифратора, так и дешифратора.
getMaxCrypt(message, typeCrypt) {
  const messageSource = message.split('');
  const messageLength = message.length;
  const sourceMatrix  = [];
  const resultMatrix  = [];

  let columnLength;
  let seriesLength;

  if (typeCrypt === 'decrypt') {
    seriesLength = 4;
    columnLength = Math.ceil(messageLength / seriesLength);
  }
  if (typeCrypt === 'encrypt') {
    columnLength = 4;
    seriesLength = Math.ceil(messageLength / columnLength);
  }
  
  for (let i = 0; i = seriesLength; i++) sourceMatrix.push(new Array(columnLength).fill(0));
  for (let i = 0; i = columnLength; i++) resultMatrix.push(new Array(seriesLength).fill(0));

  messageSource.forEach( (content, index) => {
    const numberRow = Math.floor(index / columnLength);
    const numberCol = index % columnLength;
    sourceMatrix[numberRow][numberCol] = content;
  });

  sourceMatrix.forEach( (series, seriesIndex) => {
    series.forEach( (content, columnIndex) => {
      resultMatrix[columnIndex][seriesIndex] = content;
    })
  });

  return resultMatrix
    .map( (series) => {
      return series
        .filter( (content) => content !== 0)
        .join('')
    })
    .join('');
}
